Class Store<S>

A store manages a single object which must be JSON and contain only serializable values. A store maintains its own copy of the original object so external modifications have no effect after initialization. A store provides APIs to update and query views of the internal object using the MongoDB query language. A view can be subscribed to for changes by registering listeners which are notified when their values change.
